**madside** is an in-browser IDE for retro hardware. Write 6502 / Z80 / 68000 assembly or C, build it, run it on an emulator, and debug it with breakpoints, a register view, a memory inspector, and a typed variables panel — all in the browser, with nothing to install.

It currently targets five machines:

- **Atari 8-bit** (800XL / 130XE) — builds to `.xex`, running on the Altirra core.
- **NES** — builds to an `.nes` iNES ROM, running on the jsnes core.
- **Commodore 64** — builds to a `.prg`, running on the Chips `c64-core`.
- **ZX Spectrum 48K** — builds to a `.sna`, running on the Chips `zx-core`.
- **Sega Genesis** (Mega Drive) — builds to a flat `.bin` ROM, running on the Genesis Plus GX core.

Four toolchains ship, all compiled to WebAssembly:

- **MADS** — the Mad-Assembler, a 6502 macro assembler.
- **cc65** — the cc65 C compiler with its ca65 assembler and ld65 linker, targeting the 6502 machines (NES, Atari, C64). Write C (or ca65 assembly) and it links against the bundled cc65 runtime for the active machine.
- **z88dk** — the sccz80 C compiler with the z80asm assembler, targeting the ZX Spectrum.
- **clownassembler** — a Motorola 68000 assembler (asm68k / SN-68k syntax), targeting the Sega Genesis.

C editing is backed by an in-repo **language server** (`@madside/lsp-*`): a language-agnostic core with per-dialect engines for cc65 (6502) and sccz80/z88dk (z80), giving completion, hover, go-to-definition, references, rename, and diagnostics.

Everything beyond the code editor is a **plugin**: machines, toolchains, emulators, debug adapters, panels, file converters, and visual editors. Adding a new retro platform doesn't touch the workbench — it ships as a set of plugins.

## What you can do

- Start from a **template** or pick a guided **course**, then edit, build, and run.
- Set source-level **breakpoints**, step by instruction or by frame, and watch registers, flags, memory, and typed **variables** (with a struct/array/pointer tree + watch expressions on C builds).
- Rearrange the workspace freely — every panel is **dockable** (drag, float, hide, save presets).
- Inspect machine-specific state — the NES build ships a **PPU viewer** (pattern tables + palette).
- Convert assets (images, CSV, binaries) into assembler data with the **asset pipeline**.
- Keep your work in the browser (IndexedDB), snapshot it automatically, and export/import projects as ZIP.

## Who these docs are for

- **Users** — people writing retro assembly or C in madside. Start with [Getting Started](/docs/getting-started/open/).
- **Plugin authors** — people adding machines, toolchains, panels, converters, or editors. See [Extending madside](/docs/extending/).

:::note
Learning to *program* a specific machine (Atari display lists, NES PPU, …) is the job of the interactive **courses**, not this reference. These docs cover the tool and its extension points.
:::
