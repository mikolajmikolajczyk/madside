This section covers using madside itself — independent of which machine you target. Learning to program a specific machine is the job of the interactive **courses**.

Pages (filling in incrementally):

- **Projects** — the manifest, file tree, main file, ZIP import/export, storage.
- **Editor** — autocomplete, hover, go-to-definition, breakpoints, the gutters.
- **Building** — auto-assemble, the Output panel, the source map.
- **Running** — the run controls, keyboard shortcuts, audio.
- **Debugging** — breakpoints, step vs frame, registers / flags, the memory viewer.
- **Panels** — registers, memory, output, and machine-specific viewers.
- **Asset pipeline** — converters and recipes.
- **History** — automatic and manual snapshots.
- **Templates** — starting from a bundled template.
- **Courses** — taking an interactive course.
- **Keyboard shortcuts**.

Start with [Projects](/docs/using/projects/).
